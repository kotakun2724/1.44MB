## Budget summary

- Target floppy size: **1,474,560 bytes** (1.44 MB, 3.5" HD floppy)
- Final submission `.exe`: **72,192 bytes** (70 KB)
- Headroom: **1,402,368 bytes** unused (95.1 % of the disk)
- Compression: raw ~209 KB -> UPX 72 KB (~2.9x ratio, --best --lzma)

## DCSS tiles cost breakdown (stage 11 - stage 10)

- Embedded PNG payload: 38,874 bytes across 44 tiles (in `src/assets_gen.c`, generated by `tools/gen_assets.py`).
- Win UPX delta: **+35 KB** -- PNGs are already DEFLATE-compressed, so UPX/LZMA can barely shrink them further; the embedded byte count basically passes straight through.
- Renderer code itself shrank (procedural texture generators removed); the growth is almost entirely the tile data.
- RAM at runtime: 44 tiles x 32x32x4 = ~180 KB of decoded bitmaps in the heap. Not part of the on-disk file.
- The upstream tileset folder is NOT committed (gitignored). Only `src/assets_gen.c` ships; regenerate with `make assets`.
